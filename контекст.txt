Ты решаешь задачу оптимального управления в соревновательной пошаговой
стратегической среде с неполной информацией, ограниченными ресурсами и динамически
изменяющейся картой. Задача относится к классу многокритериальной оптимизации с
элементами теории игр, где необходимо одновременно максимизировать прибыль,
контролировать риски и учитывать действия противников.

КОНТЕКСТ ЗАДАЧИ
Среда представляет собой дискретное пространство (двумерная сетка), в котором агент
управляет набором связанных между собой узлов (плантаций). Эти узлы формируют сеть с
центральным управляющим элементом (ЦУ), потеря которого приводит к полному обнулению
текущего прогресса.
Каждый ход агент получает состояние среды и должен принять набор решений,
ограниченных:
•

числом доступных управляемых узлов

•

радиусами взаимодействия

•

деградацией эффективности при передаче команд

•

лимитами на количество сущностей

Система работает в условиях:
•

конкуренции (другие агенты)

•

стохастических событий (катаклизмы)

•

частичной информации (ограниченный радиус обзора)

ЦЕЛЬ ОПТИМИЗАЦИИ
Цель — максимизировать суммарную полезность (очки), получаемую за последовательность
раундов, при этом:
1. Минимизировать вероятность катастрофических потерь (loss of control event)
2. Обеспечить устойчивый поток дохода (терраформация)
3. Использовать возможности для скачкообразного прироста (атаки, бобры)
4. Учитывать долгосрочную перспективу (несколько раундов)
Это не задача максимизации одного показателя, а задача балансировки:
•

доход vs риск

•

экспансия vs контроль

•

агрессия vs стабильность

МОДЕЛЬ СОСТОЯНИЯ

Состояние на каждом шаге включает:
1. Геометрию карты
2. Позиции собственных узлов
3. Позиции наблюдаемых вражеских узлов
4. Прогресс изменения среды (терраформация)
5. Временные события (катаклизмы)
6. Ограничения (лимиты, апгрейды)
При этом важно:
•

информация неполная (fog of war)

•

часть будущего известна через прогнозы (метeoForecasts)

•

система динамически изменяется

МОДЕЛЬ ДЕЙСТВИЙ
Каждый ход агент формирует набор команд, каждая из которых:
•

проходит через цепочку узлов

•

теряет эффективность на каждом шаге

•

влияет на состояние среды

Тип действия определяется целью:
•

расширение (строительство)

•

поддержка (ремонт)

•

агрессия (диверсия)

•

взаимодействие с нейтральными объектами (бобры)

Также есть мета-действия:
•

апгрейды (изменяют глобальные параметры)

•

перенос центра (меняет топологию сети)

ОГРАНИЧЕНИЯ
Задача осложняется следующими ограничениями:
1. Топологические:
•

управление возможно только в связной компоненте

•

разрыв сети = потеря функциональности

2. Ресурсные:

•

лимит на количество узлов

•

ограниченное число действий в ход

3. Геометрические:
•

радиусы действия задают локальность

•

эффективность падает с расстоянием

4. Временные:
•

каждое решение влияет на будущие состояния

•

есть события с отложенным эффектом (деградация клеток)

НЕОПРЕДЕЛЕННОСТЬ
Система содержит:
•

случайные события (бури, землетрясения)

•

неизвестные действия противников

•

конфликты за ресурсы (клетки, бобры)

Это требует:
•

оценки вероятностей

•

построения устойчивых стратегий

•

избегания решений с высоким риском отказа

СТРУКТУРА ОПТИМАЛЬНОГО РЕШЕНИЯ
Лучшее решение должно быть построено как:
1. Иерархическая система:
o

уровень выживания (safety layer)

o

уровень экономики (income layer)

o

уровень тактики (tactical layer)

2. Система приоритетов:
o

критические угрозы имеют абсолютный приоритет

o

долгосрочная выгода важнее краткосрочной, если нет риска отставания

3. Предиктивная модель:
o

оценка будущих состояний

o

учет деградации, завершения терраформации, перемещения угроз

4. Локальная оптимизация:

o

решения принимаются с учетом ближайших выгод и ограничений

5. Глобальная согласованность:
o

все действия должны поддерживать целостность сети

КРИТЕРИИ КАЧЕСТВА РЕШЕНИЯ
Решение считается оптимальным, если:
•

отсутствуют катастрофические ошибки (потеря ЦУ, разрыв сети)

•

поддерживается высокий темп генерации очков

•

эффективно используются возможности для агрессии

•

минимизируются потери ресурсов

•

соблюдается баланс между расширением и защитой

ОЖИДАЕМЫЙ ПОДХОД
При построении решения необходимо:
•

формализовать функцию полезности

•

учитывать ограничения как жесткие (hard constraints)

•

использовать эвристики для оценки состояний

•

минимизировать зависимость от случайности

•

обеспечивать устойчивость к действиям противников

ИНТЕРПРЕТАЦИЯ ЗАДАЧИ
Это задача не про поиск одного идеального действия, а про построение системы, которая:
•

адаптируется к изменяющимся условиям

•

избегает проигрышных сценариев

•

стабильно извлекает максимум из доступной информации

Решение должно быть не жадным, а стратегически устойчивым.
